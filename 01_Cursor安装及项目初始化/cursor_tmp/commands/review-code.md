# 代码审查清单

## 概述
用于进行全面代码审查的完整检查清单，以确保代码的质量、安全性和可维护性。

## 审查分类

### 功能性
- [ ] 代码实现了它应该实现的功能
- [ ] 边界情况（edge cases）已得到处理
- [ ] 错误处理合理且充分
- [ ] 没有明显的 bug 或逻辑错误

### 代码质量
- [ ] 代码可读性好，结构清晰
- [ ] 函数体积小、职责单一（专注）
- [ ] 变量命名具有描述性且清晰
- [ ] 不存在明显的代码重复
- [ ] 遵循项目既定的编码规范和约定

### 安全性
- [ ] 没有明显的安全漏洞
- [ ] 存在输入验证和清理机制
- [ ] 敏感数据得到适当的处理和保护
- [ ] 不存在硬编码的密钥、密码或凭证
